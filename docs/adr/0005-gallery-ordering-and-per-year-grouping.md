# Galleries sort by year then a manual `order`, and group into per-year sections

Both collections gain an optional `order: z.number().int().optional()` attribute. Each gallery (paintings and ceramics) sorts Artworks by **year descending**, then by **`order` ascending**; Artworks without an `order` fall to the end of their year in **slug order** (no other fallback). Each gallery is then split into per-year sections, each introduced by an `<h2>Selected Works {year}</h2>` (identical copy on both galleries), newest year first.

`order` is deliberately scoped **per year**: because year is the primary sort key, an `order` value is only ever compared against other Artworks of the same year. Jo numbers within a year and restarts each year; duplicate `order` values inside one year fall back to slug order between them, and `order` is meaningless across years. This gives manual control of sequence within a year without forcing a global renumber, and keeps each `Selected Works` section internally coherent (a year header can never repeat).

The comparator is intentionally mixed-direction (year ↓, order ↑): newest work surfaces first, but within a year the lowest `order` reads first, matching how a curator lists a set. Consequence: the Ceramics masonry grid is now one grid per year-section rather than one continuous grid, so its rhythm resets at each year boundary — accepted, since `order` and year headers were wanted on both galleries. The dangling separator in the Artwork detail meta line (`{year} · {medium}` rendering `2024 · ` when medium is absent, e.g. every ceramic) is fixed at the same time to match the already-guarded gallery card: `{year}{medium && ` · ${medium}`}`.
